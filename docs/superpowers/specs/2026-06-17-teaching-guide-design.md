# PhamaCount 教學指引設計文件

## 1. 功能概述

為協助新手使用者快速上手 PhamaCount 藥局智能清點系統，提供互動式教學指引功能。透過角落的資訊按鈕 (i) 點擊彈出內嵌視窗教學，使用者可以逐步學習系統的主要功能操作。

## 2. 目標使用者

- 新手藥局人員：首次使用系統的使用者
- 需要快速了解特定功能操作的使用者
- 所有需要複習操作流程的使用者

## 3. 使用者故事

### 3.1 基本使用者故事
- 作為新手使用者，我想要有一個引導教學來學習如何使用條碼掃描功能，這樣我就能快速上手而不需要 constantemente 請教同事。
- 作為使用者，當我不確定某個功能如何操作時，我想要點擊相應功能區域的資訊按鈕來查看該功能的使用說明，這樣我就能獨立解決問題。
- 作為使用者，我想要在教學視窗中可以上下頁切換學習內容，這樣我就能按照自己的節奏學習。

## 4. 功能需求

### 4.1 核心功能
1. **教學入口**：在 UI 角落顯示資訊按鈕 (i)
2. **教學彈窗**：點擊按鈕後彸出 modal 教學視窗
3. **頁面切換**：教學視窗支援上一頁/下一頁導覽
4. **背景遮罩**：彈窗時背景變暗以聚焦教學內容
5. **關閉機制**：提供關閉按鈕或點擊遮罩關閉彈窗
6. **情境教學**：在不同功能頁面顯示相應功能的教學按鈕

### 4.2 教學內容規劃
| 教學模組 | 對應頁面 | 教學內容 |
|----------|----------|----------|
| 系統概覽 | 首頁 | 系統介紹、主要功能模組說明 |
| 條碼掃描 | 掃描頁面 | 如何使用條碼掃描、即時比對、跨頁搜尋 |
| 匯入功能 | 匯入頁面 | 如何上傳CSV清單、格式要求、匯入流程 |
| 拍照留存 | 掃描頁面 | 如何調用相機拍照、上傳證據照片 |
| 異常處理 | 掃描頁面/總結頁面 | 如何識別處理數量不符項目、異常覆核流程 |
| 報告匯出 | 總結頁面 | 如何匯出CSV報告、查看視覺化進度概覽 |

## 5. 非功能需求

### 5.1 效能需求
- 教學彈窗開啟延遲 < 300ms
- 頁面切換動畫流暢 (60fps)
- 不影響主要頁面載入效能

### 5.2 相容性需求
- 支援現代瀏覽器 (Chrome, Safari, Firefox, Edge)
- 適配手機和平板設備 (RWD)
- 與現有 Next.js 16.2.9 和 React 19.2.4 相容

### 5.3 安全性需求
- 教學內容不會洩漏敏感資訊
- 不會影響現有的認證和授權機制
- 所有互動都在前端處理，不涉及後端 API 呼叫

### 5.4 可維護性需求
- 教學內容易於更新和維護
- 組件結構清晰，易於擴展新教學模組
- 程式碼遵循專案現有的 TypeScript 標準

## 6. 系統架構

### 6.1 整體架構

- **樣式**：Tailwind CSS (完全符合專案現有設計規範)
- **動畫**：CSS 變換和過渡 (符合AGENTS.md中指定的動效要求)
- **類型安全**：TypeScript 介面定義

## 7. 組件設計

### 7.1 教學管理核心 (TeachingCore.tsx)
```typescript
interface TeachingState {
  isOpen: boolean;
  currentStep: number;
  totalSteps: number;
  teachingModule: TeachingModuleType | null;
}

interface TeachingContextType {
  state: TeachingState;
  openTeaching: (module: TeachingModuleType) => void;
  closeTeaching: () => void;
  nextStep: () => void;
  prevStep: () => void;
  setTeachingModule: (module: TeachingModuleType) => void;
}
```

### 7.2 教學入口按鈕 (TeachingButton.tsx)
- 位置：固定在視窗右下角 (或特定功能區域內)
- 樣式：符合AGENTS.md中按鈕設計規範
  - 圓角：rounded-full
  - 顏色：背景使用卡片表面色 (#162a56)，圖標使用強調色 (#00f2fe)
  - 懸停效果：背景色微調 + 發光外框
  - 按下效果：scale-95 動態微縮

### 7.3 教學彈窗容器 (TeachingModal.tsx)
- 外層容器：固定定位，覆蓋全螢幕
- 背景遮罩：rgba(0,0,0,0.5) 深色半透明 (符合科技暗黑主題)
- 內容容器：
  - 寬度：響應式 (手機 90%，平板 70%，桌面 500px)
  - 最大高度：視窗 80%
  - 背景：卡片表面色 (#162a56) + backdrop-blur-md
  - 圓角：rounded-2xl
  - 陰影：shadow-2xl (符合科技感發光特效)
  - 關閉按鈕：右上角 X 按鈕，符合按鈕設計規範
  - 導覽區域：下方固定導覽列
    - 上一頁按鈕：左側
    - 頁碼顯示：中間 (當前頁數 / 總頁數)
    - 下一頁按鈕：右側

### 7.4 教學內容呈現
- 標題：使用強調色 (#00f2fe) 和適當字重
- 內容文字：淺灰色 (#e2e8f0) 確保可讀性
- 圖示：使用 lucide-react 圖標庫，顏色為強調色
- 範例展示：適當使用框線和底色區分操作範例

## 8. 資料流程

### 8.1 教學啟動流程
```
使用者點擊教學按鈕
        ↓
點擊事件處理器
        ↓
更新教學狀態 (isOpen: true, 設定當前教學模組)
        ↓
渲染教學彈窗組件
        ↓
載入對應教學模組的第一步內容
        ↓
顯示背景遮罩和教學內容
```

### 8.2 頁面切換流程
```
使用者點擊「下一頁」按鈕
        ↓
點擊事件處理器
        ↓
更新當前步驟索引 (currentStep++)
        ↓
檢查是否為最後一頁
        ↓
如果不是最後一頁：載入下一步內容
        ↓
如果是最後一頁：禁用「下一頁」按鈕
        ↓
重新渲染教學內容區域
```

### 8.3 教學關閉流程
```
使用者點擊關閉按鈕或點擊遮罩
        ↓
點擊事件處理器
        ↓
更新教學狀態 (isOpen: false)
        ↓
卸載教學彈窗組件
        ↓
恢復正常頁面互動
```

## 9. UI/UX 設計規範 (符合 AGENTS.md)

### 9.1 主題色調 (Dark Mode 科技風格)
- 背景：#07142b (深藍底色)
- 卡片表面：#162a56 搭配 backdrop-blur-md (毛玻璃特效)
- 強調/成功：#00f2fe (極光藍發光特效)
- 警告/缺失：#ff4b5c (霓虹紅)
- 文字主要顏色：#f8fafc (近白)
- 次要文字顏色：#cbd5e1 (淺灰藍)

### 9.2 元件設計
#### 按鈕設計
- 基本樣式：rounded-xl (主要按鈕) 或 rounded-full (圓形按鈕)
- 焦點狀態：shadow-[0_0_15px_rgba(0,242,254,0.5)] 發光外框
- 按下狀態：active:scale-95 動態微縮
- 禁用狀態：opacity-50 + cursor-not-allowed

#### 輸入框 (如適用)
- 焦點狀態：shadow-[0_0_15px_rgba(0,242,254,0.5)] 發光外框
- 基本樣式：border-none + bg-transparent + text-white + placeholder-gray-400

### 9.3 動畫與過渡
- 彈窗進出：淡入淡出 (opacity 0 → 1) + 輕微縮放 (scale 0.95 → 1)
- 頁面切換：水平滑動過渡 (transform: translateX)
- 按鈕懸停：顏色漸移 + 輕微提升 (translate-y-[-2px])
- 所有過渡duration：200ms - 300ms，使用 ease-out 曲線

### 9.4 響應式設計
- 手機寬度 (< 640px)：教學彈窗寬度 90%，邊距 5%
- 平板寬度 (640px - 1024px)：教學彈窗寬度 70%，邊距 15%
- 桌面寬度 (> 1024px)：教學彈窗寬度 500px，水平居中
- 所有內容使用相對單位 (rem, %) 而非固定像素

## 10. 錯誤處理

### 10.1 前端錯誤
- 教學資料載入失敗：顯示預設錯誤訊息並提供重試按鈕
- 教學步驟索引超出範圍：自動重置為第一頁或最後一頁
- DOM 操作失敗：優雅降級，不影響主要功能

### 10.2 使用者錯誤
- 重複快速點擊：使用防抖 (debounce) 避免狀態競爭
- 無效操作：提供視覺回饋 (輕微震動或顏色閃爍) 但不中斷流程

### 10.3 邊界情況
- 首次使用 vs 回覆使用：可考慮加入「不再顯示」選項 (未來擴展)
- 教學內容為空：顯示「此功能暫無教學內容」訊息
- 網路中斷：完全前端實作不受影響

## 14. 未來擴展可能性

1. **個人化教學**：根據使用者角色或權限顯示不同教學內容
2. **進度追蹤**：記錄使用者完成的教學模組，避免重複顯示
3. **多語言支援**：教學內容支援多種語言切換
4. **視頻教學**：在教學內容中嵌入短影片示範
5. **互動式引導**：不僅是靜態教學，還能引導使用者實際操作特定步驟
6. **AI 助教**：結合現有的 Google Generative AI 提供智能答疑
